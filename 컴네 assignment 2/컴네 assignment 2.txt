입력 (request line) 예시 
1. initial line (ex) GET /path/to/file/index.html HTTP/1.0 ) -> 이것도 GET /... HTTP/1.0의 두 spacebar 등 여러 공백 위치 가능
2. 0 이상 header lines -> 여러 줄 가능해서 CRLF로 끝나야 한다고 하는 것으로 받아들이면 됨 (3번).
-> header name은 대소문자 구분 안 함(value는 주의), " : " 와 value 사이 spacebar, tab 개수 제한 없음, header 여러 line이더라도 \n하고 spacebar나 tab 하고 적으면 기존 줄의 내용이 이어진 것.
-> header 종류 몇 개 있음 -> 과제에 필요 여부는 아직 확인 안 함.
3. usually CRLF (\r\n) (gracefully handle just LF(\n)
4. optional msg body

출력 명령어 (response line) 예시
1. 정상 (HTTP/1.0 200 OK) / 에러 (HTTP/1.0 404 Not Found)
2. 0 이상 header lines
3. usually CRLF (\r\n) (gracefully handle just LF(\n)
4. optional msg body


appendix (구현 시 주의사항)
- “HTTP/1.0” 필수, 없으면 에러 출력.
- GET만 구현하면 됨.
- GET 뒤의 host와 2번 줄의 header의 host가 다르면 400 Bad request error 출력.
- 에러 메시지 : HTTP/1.0 400 Bad Request\r\n (empty host header field의 경우)
- proxy가 request valid 여부 확인해서 400 Bad Request 보내고, server와 connect하면 안 됨 (= server의 400 이 아니라는 의미).
- 밀리초 단위 실패 가능 (proxy 거쳐서 data 시간 차이) -> 발생하면 report.pdf에 작성 추천.
- transaction 끝나고 client & server 연결 끊어야 함.
- testcase : host : http://www.google.com 로 http 붙은 것 고정임 (그냥 바로 www. 이 안 나온다는 의미)
- proxy ctrl + c로 끝내면 됨 = while 종료 조건 없음.
